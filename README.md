3. Піни
   
   2.1 PA1 - Для реалізації різних візуальних ефектів система динамічно змінює режим роботи піна PA1             залежно від стану FSM:
                                                                                                                 Режим GPIO_MODE_OUTPUT_PP використовується у станах IDLE, BLINK_SLOW, BLINK_FAST та PANIC.                Частота мигання світлодіод керується безспосередньо перериваннями заданого таймера TIM2.             
       Режим GPIO_MODE_AF_PP використовується виключно у стані BREATHE. Керування піном передається              апаратному таймеру TIM2 для генерації ШІМ-сигналу (PWM). Яскравість змінюється автоматично завдяки        роботі DMA , який переносить значення з гамма - коригованої кривої.
                                                                                                                 Зміна конфігурація піна відбувається в функціях GPIO_LED_OUTPUT та HAL_TIM_MspPostInit.

   2.2 PA2 - цей пін відповідає за роботу з кнопкою. Працює в режимі GPIO_MODE_IT_FALLING який фіксує            натискання кнопки в момент падіння  логічного рівеня з 1 до 0. Для стабілізації сигналу                   активовано підтяжку GPIO_PULLUP, яка дозволяє завжди тримати пін в 1 за відсутності натискання.                                           
       Пін підтримує EXTI преривання так як це пін PA2 я використовував EXTI2_IRQn з базовим преорітетом         5. Як показала практика такий преорітет є ідеальним для роботи переривання.

   2.3 PA9 - UART_TX пін використовується для виводу повідомлень в Serial monitor через UART інтерфейс.
   
   2.4 PA10 - UART_RX пін використовується для отримання данних з Serial monitor через UART інтерфейс.           Зокрема для отримання реалізованих команд(status, tasks, drop_to_idle, panic, reset_stats).

5. Архітектура системи
        
   5.1 Задачі - Було створено 5 задач:
       ButtonTask - Отримує стан GPIO, виконує програмний debounce, детектує типи натискань (short, long,        double). Надсилає сформовані події в чергу Queue_FSM.
                                                                                                                 DispatchTask - Отримує події з черги Queue_FSM, оновлює стан кінцевого автомата (FSM) та розсилає         нотифікації/дані іншим задачам. Це єдине місце, де знаходиться логіка FSM.                                                                                                                                          LedTask - Керує світлодіодом виключно на основі отриманих від DispatchTask команд. Реалізація PWM         + DMA знаходиться тут.                                                                                                                                                                                              UartTask - Єдина задача, яка має право працювати з UART. Отримує готові рядки через чергу                 Queue_UART та виводить їх у термінал.                                                                                                                                                                               StatsTask - Збирає runtime статистику системи та ініціює ї ї вивід раз на 10 секунд.
